# Backlog Refinement Checklist

## Pre-Refinement Checks

- [ ] Identified stories with `detail_level: placeholder` or `outlined`
- [ ] Grouped stories by epic for user selection
- [ ] Loaded relevant context documents (PRD, Architecture, UX)

## Story Refinement Checks (per story)

### Story Structure
- [ ] Story has clear title
- [ ] Story has `detail_level: ready`
- [ ] Story has point estimate (Fibonacci: 1, 2, 3, 5, 8, 13, 21)
- [ ] Story follows user story format (As a/I want/So that)

### Acceptance Criteria
- [ ] Each AC uses Given/When/Then format
- [ ] ACs are specific and testable
- [ ] ACs cover happy path
- [ ] ACs cover edge cases
- [ ] ACs cover error conditions

### Story Quality
- [ ] Story is appropriately sized (3-8 points ideal)
- [ ] Story can be completed by single developer
- [ ] Story has no dependencies on future stories
- [ ] Story delivers clear user value

### Context Integration
- [ ] Story aligns with PRD requirements
- [ ] Story follows Architecture patterns
- [ ] Story incorporates UX requirements (if UI)

## Post-Refinement Checks

- [ ] Epic file updated with refined story
- [ ] `detail_level` changed to `ready`
- [ ] Points estimate confirmed
- [ ] Full user story and ACs added
- [ ] No duplicate or conflicting stories

## Completion

- [ ] User confirmed all refinements
- [ ] User informed about sprint-planning workflow
